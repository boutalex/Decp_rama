general_process.GlobalProcess module¶

class general_process.GlobalProcess.GlobalProcess(data_format='2022')[source]¶

Bases : object

La classe GlobalProcess est une classe qui définit les étapes de traitement une fois toutes les étapes pour toutes les sources effectuées : création des variables de la classe (__init__), fusion des sources dans un seul DataFrame (merge_all), suppression des doublons (drop_duplicate) et l’exportation des données en json pour publication (export).

drop_by_date_2024()[source]¶

Supprime les lignes ne respectant pas les critères de date. Si le format suivi est de 2022, les champs “dateNotification” et “dateDebutExecution” doivent être supérieurs au 01/01/24. Si le format suivi est de 2019, ces champs doivent être inférieurs au 01/01/24.

drop_duplicate()[source]¶

L’étape drop_duplicate supprime les duplicats purs après avoir supprimé les espaces et convertis l’ensemble du DataFrame en string.

export()[source]¶
fix_all()[source]¶

Étape fix all qui permet l’uniformisation du DataFrame.

merge_all()[source]¶

Étape merge all qui permet la fusion des DataFrames de chacune des sources en un seul.

upload_datagouv()[source]¶

upload_s3()[source]¶

Cette fonction exporte decpv2 sur le S3 decp.

general_process.SourceProcess module¶

class general_process.SourceProcess.SourceProcess(key, data_format)[source]¶

Bases : object

La classe SourceProcess est une classe abstraite qui sert de parent à chaque classe enfant de sources. Elle sert à définir le cas général des étapes de traitement d’une source : création des variables de classe (__init__), nettoyage des dossiers de la source (_clean_metadata_folder), récupération des URLs (_url_init), get, convert et fix.

check(jsonData, xml_path) → bool[source]¶

Fonction qui prend en paramètre une donnée json ou xml et vérifiant, grâce à un schéma, que la donnée est valide.

Paramètres:

  • jsonData – donnée json en entrée (None si on vérifie un fichier xml)

  • xml_path – chemin du fichier xml en entrée (None si on vérifie un fichier json)

check_date_file(url: list, title: list, new_ressources: dict, old_ressources: dict) → list[source]¶

Fonction vérifiant si la date de dernière modification des fichiers ressources dans les metadatas est strictement antérieure à la date de dernière modification.

Paramètres:

  • url – la liste contenant les liens pour télécharger les fichiers

  • new_ressources – dictionnaire correspondant au champ « resources » dans le fichier metadata de la source

  • old_ressources – dictionnaire correspondant au champ « resources » dans le fichier old_metadata de la source

clean()[source]¶

Cette fonction extrait les dictionnaires des fichiers (suivant le format 2022) pour qu’ils puissent être nettoyé. Grâce à la fonction tri_format, un tri est effectué sur ces dictionnaires pour séparer les bons marchés, des mauvais.

comment()[source]¶

Cette étape permet de marquer les cases vides du dataframe avec un indicateur (« MD », ou « CDL »).

concession_mark_fields(df: DataFrame) → DataFrame[source]¶

Commente toutes les colonnes du dataframe en appelant es fonctions mark_mandatory_field et mark_optional_field. Le dataframe doit contenir que des concessions.

Paramètres:

df – dataframe dans lequel s’effectue les modifications

convert()[source]¶

Étape de conversion des fichiers qui concatène les fichiers présents dans {self.source} dans un seul DataFrame. Elle utilise le dictionnaire des marchés/concessions valides de chaque fichier pour le convertir en un dataframe. L’ensemble des dataframes est stocké dans une liste.

Paramètres:

col_name – colonne où s’effectue le changement

convert_prestataire()[source]¶

Étape de conversion des fichiers qui supprime les “ et concatène les fichiers présents dans {self.source} dans un seul DataFrame

create_metadata_file(n: int) → tuple[list, list][source]¶

Fonction réalisant le téléchargement des métadatas, la copie des fichiers métadatas et la création des listes contenant les titres et les urls des fichiers.

Paramètres:

date_after_2024(record: dict) → bool[source]¶

La fonction prend en entrée un dictionnaire et renvoie un booléeen.Les dates postérieures à 2024 doivent être de la forme Y-M-J pour les colonnes date et date_de_publication.

Paramètres:

record – dictionnaire dans lequel se fait la vérification

date_before_2024(record: dict, nature: str) → bool[source]¶

La fonction vérifie que les dates contenues dans un marché/une concession sont postérieures à 2024 et sont de la forme Y-M-J pour les colonnes date et date_de_publication.

Paramètres:

  • record – dictionnaire dans lequel se fait la vérification

  • nature – il s’agit soit d’une concession, soit d’un marché

date_norm(datestr: str)[source]¶

Permet de modifier le format d’une date. Plus précsément, les “+” sont remplacés par des “-”

download_without_metadata()[source]¶

Fonction téléchargeant un fichier n’ayant pas de clé api. Par conséquent, le téléchargement s’effectue grâce à l’url dans l’attribut url_source

Étape get qui permet le lavage du dossier sources/{self.source} et la récupération de l’ensemble des fichiers présents sur chaque url.

mark_mandatory_field(df: DataFrame, field_name: str) → DataFrame[source]¶

  • filed_name – nom d’une colonne qui est obligatoire

mark_optional_field(df: DataFrame, field_name: str) → DataFrame[source]¶

Complète les lignes vides de la colonne « field_name » avec le string “CDL”

Paramètres:

  • filed_name – nom d’une colonne qui est optionnel

tri_format(dico: dict, file_name: str) → None[source]¶

Cette fonction permet de vérifier la structure du dictionnaire fournit en entrée. Si le schéma n’est pas respecté, les marchés et concessions correctes sont stockés dans des listes et des fichiers pour une future utilisation. Il en est de même pour les marchés et les concessions incorrectes.

Paramètres:

  • dico – il s’agit d’un dictionnaire avec 2 clés: “marchés” et “contrat-concession”

  • file_name – nom du fichier où se trouve le dictionnaire dico

validateJson(jsonData: dict, jsonScheme: dict) → bool[source]¶

Fonction vérifiant si le fichier jsn « jsonData » respecte le schéma spécifié dans le schéma en paramètre « jsonScheme ».

Paramètres:

  • jsonData – dictionnaire qui va être vérifié par le validateur

  • jsonScheme – schéma à respecter

validateXml(xml_path: str, xsd_path: str) → bool[source]¶

  • xsd_path – chemin du schéma
